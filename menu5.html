도커란? 
도커는 2013년 3월 출시된 오픈소스 컨테이너 프로젝트이다.
기본적으로 "컨테이너"라고 부르는 애플리케이션 자동화 기술을 좀 더 쉽게 사용할 수 있도록 하기 위해 만들어졌다.
초기에는 LXC(Linux X Container)를 기반으로 구현
버전 0.9부터는 libcontainer를 개발하여 사용

특징
컨테이너 안에서도 호스트와 거의 동일한 성능
별도의 OS 설치할 필요가 없음, 용량부담 ↓
앱을 만들때 하드웨어가 달라져도 시스템설정 유지가능
이미지를 설치하기 이전에는 내용 확인불가능 (보안상 취약점)
개발환경을 구축하는데 걸리는 시간 단축
격리된 환경을 활용해 실험적인 개발가능 + 비용절감

이미지 
이미지는 서비스에 필요한 프로그램, 라이브러리, 소스 코드 등을 묶은 파일을 뜻한다.
운영체제에 비유하면 이미지는 실행파일, 컨테이너는 프로세스.
컨테이너로 구축한 환경을 오픈소스 저장소에 이미지로 배포하게되면
타 유저가 해당 이미지를 다운로드받아 자신의 도커엔진으로 새로운 컨테이너를 구동할 수 있다.
도커는 리눅스 환경에서만 구동되는데, 리눅스 터미널에서 도커 엔진을 통해 이미지를 검색한다.

- 유니온 파일 시스템 형식 (aufs, btrfs, devicemapper)
도커는 이미지의 바뀐 부분만을 이미지로 생성, 실행시 베이스 이미지와 합쳐서 실행한다.
각 이미지는 의존 관계를 형성한다.

Docker vs VM 
가상화기술은 하이퍼바이저라는 기술이 반드시 있어야한다.
하이퍼바이저는 하나의 컴퓨터에서 여러개의 OS를 사용할 수 있게 도와주는 기술로
가상화 환경은 가장 밑단에 깔린 호스트 OS를 공유한다.

도커는 하이퍼바이저와 달리 게스트 OS를 두지않고 호스트 OS 커널(리눅스의 핵심요소)을 바로 사용한다.
하이퍼바이저대신 도커 엔진이 올라가 호스트OS와 여러 애플리케이션을 연결해주는 역할을 한다.

도커를 사용하면 가상화보다는 내부에서 더 작은 일을 처리하고, 애플리케이션을 좀 더 빠르고 효율적으로 실행시킬 수 있다.
